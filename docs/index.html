Bienvenue sur Clipr

A quoi sert Clipr ?

Clipr vous aide a decouper des videos longues (interviews, reportages, conferences...) en extraits thematiques.

Deux facons de travailler :

  • Analyse IA — L'intelligence artificielle ecoute la video, transcrit les paroles et cree automatiquement des chapitres thematiques. Ideal pour gagner du temps sur des videos longues.
  • Decoupe manuelle — Vous creez vos propres segments sur la timeline, a la main. Controle total.
Vous pouvez meme lancer une analyse IA sur un projet et travailler manuellement sur un autre en attendant !

En resume

  • Jusqu'a 6 projets en parallele
  • L'IA travaille en arriere-plan : pas besoin de rester sur la page
  • Chaque utilisateur a son propre espace
  • Possibilite de partager un projet avec des collegues
  • Export en video MP4, texte ou fichier projet

Analyse IA

Comment ca marche ?

L'IA va ecouter votre video, transcrire tout ce qui est dit, puis decouper automatiquement le contenu en chapitres thematiques avec des titres.

1 Importez une ou plusieurs videos dans votre projet

Choisissez entre l'analyse IA automatique ou la decoupe manuelle

2 Choisissez le mode Analyse IA

3 Dans le panneau a droite, vous pouvez ajuster :

  • Consignes de decoupe — Decrivez ce que vous attendez. Par exemple : "Concentre-toi sur les anecdotes personnelles" ou "Cree un chapitre par theme aborde"
  • Vocabulaire — Si votre video contient des mots rares, du patois, des noms de lieux... ajoutez-les ici pour que la transcription soit plus fidele
Le panneau IA avec les options de modele, langue, consignes et vocabulaire

4 Cliquez Lancer l'analyse

Pas besoin de rester sur la page ! L'analyse tourne en arriere-plan. Vous pouvez revenir a l'accueil, travailler sur un autre projet, ou meme fermer votre navigateur. Vos resultats seront la quand vous reviendrez.

Que se passe-t-il pendant l'analyse ?

Extraction du son
→
Transcription des paroles
→
Decoupe en chapitres
→
Pret !

Si vous restez sur la page, vous verrez la progression en temps reel. Sinon, la carte du projet passera au statut Termine sur la page d'accueil.

A propos du vocabulaire

Le champ Vocabulaire Whisper aide l'IA a mieux reconnaitre les mots difficiles : noms propres, termes techniques, patois, noms de lieux...

Par defaut, un vocabulaire lie au patrimoine culturel de Vendee et Bretagne est pre-rempli (artisanat, musique traditionnelle, gastronomie, etc.). Vous pouvez le modifier librement selon votre sujet.

L'editeur

L'interface d'edition

Une fois vos segments crees (par l'IA ou manuellement), vous accedez a l'editeur. Il se compose de :

  • Le lecteur video — En haut, pour visionner et naviguer dans la video. Cliquez sur un segment pour le previsualiser.
  • La timeline — La forme d'onde audio avec vos segments en couleur. Vous pouvez les redimensionner en tirant les bords, et les deplacer.
  • La liste des segments — A droite, avec les titres, durees et timecodes. Cliquez pour selectionner, double-cliquez sur le titre pour le renommer.

L'IA est occupee ?

L'IA ne peut analyser qu'un seul projet a la fois (pour tous les utilisateurs). Si quelqu'un d'autre est en train d'utiliser l'IA, vous verrez un message vous indiquant qui l'utilise et sur quel projet.

En attendant, vous pouvez :

  • Travailler en mode manuel sur un autre projet
  • Modifier les segments d'un projet deja analyse
  • Exporter vos projets termines

L'IA se libere automatiquement quand l'analyse est terminee (ou apres 30 minutes maximum en cas de probleme).

Transcription audio/video

Transcrire un fichier

L'outil de transcription convertit vos fichiers audio ou video en texte grace a Whisper (IA de reconnaissance vocale).

  1. Depuis l'accueil, cliquez sur Transcrire un media ou choisissez + Nouveau → Transcription audio/video
  2. Glissez un fichier audio (WAV, MP3, FLAC, M4A) ou video (MP4, MOV, AVI, MKV, MTS)
  3. Configurez le modele Whisper et la langue
  4. Cliquez Lancer la transcription
Les fichiers video sont automatiquement convertis : l'audio est extrait avant la transcription. Les fichiers MTS sont remuxes en MP4 pour la lecture dans le navigateur.

Identification des locuteurs

Apres la transcription, l'IA identifie automatiquement qui parle dans l'enregistrement :

  • Les locuteurs sont detectes par analyse vocale (SpeechBrain / pyannote)
  • Le nombre de speakers est auto-detecte (ou configurable)
  • Les noms sont identifies via Ollama si possible, sinon "Intervenant" / "Intervieweur"
  • Cliquez sur un nom de speaker pour le renommer — ca change partout dans la transcription

Resultats et export

  • La transcription s'affiche en temps reel pendant le traitement
  • Telechargez en .txt (texte brut avec timestamps) ou .srt (sous-titres)
  • Les exports incluent les noms des locuteurs
  • Les projets sont accessibles depuis l'historique et la page d'accueil

Gros fichiers

Les fichiers de plus de 90 Mo sont automatiquement decoupes en morceaux (chunked upload) pour passer a travers le tunnel Cloudflare. Une barre de progression s'affiche pendant l'upload.

Transcription linguistique

A quoi ca sert ?

Cet outil est concu pour documenter les langues vernaculaires (patois, dialectes locaux). Il traite des enregistrements ou un meneur dit une phrase en francais, puis plusieurs personnes la repetent dans leur langue vernaculaire.

L'outil produit pour chaque phrase :

  • Le texte francais (transcription Whisper)
  • La transcription phonetique IPA de chaque variante vernaculaire (Allosaurus)
  • Un extrait audio par variante (ecoutable dans l'interface)
  • L'identification du locuteur (renommable)

Comment l'utiliser

  1. Depuis l'accueil, cliquez sur Transcription linguistique (carte verte)
  2. Glissez votre fichier audio
  3. Indiquez le nombre de locuteurs (meneur + intervenants)
  4. Lancez l'analyse — le pipeline s'execute automatiquement
Le meneur est automatiquement identifie comme le premier speaker du fichier. Vous pouvez le changer via le selecteur dans les resultats.

Editer les resultats

  • Texte francais : cliquez dessus pour modifier (correction Whisper)
  • IPA : cliquez sur la transcription phonetique pour la corriger manuellement
  • Reset IPA : bouton pour revenir a la version Allosaurus originale
  • Renommer un speaker : cliquez sur le nom du locuteur
  • Ecouter : bouton play a cote de chaque variante

Export

Exportez vos resultats en JSON (structure complete) ou CSV (tableau : sequence, francais, locuteur, IPA, debut, fin).

Identification des locuteurs

Comment ca marche

La diarisation identifie automatiquement les differentes voix dans un enregistrement. Elle est utilisee dans les deux outils de transcription :

  • Transcription audio/video : identifie "Intervenant" et "Intervieweur" (ou les prenoms si detectes)
  • Transcription linguistique : identifie le meneur et chaque personne qui repete en vernaculaire

La technologie utilisee est pyannote (via WhisperX), le modele de diarisation le plus precis disponible.

Renommer les locuteurs

Les locuteurs sont nommes automatiquement (SPEAKER_00, SPEAKER_01...). Pour les renommer :

  1. Cliquez sur le nom du locuteur dans la transcription
  2. Tapez le nouveau nom
  3. Le renommage s'applique a tous les segments de ce locuteur
  4. Les exports (.txt, .srt, JSON, CSV) refletent les nouveaux noms

Qu'est-ce que Clipr ?

Clipr est une plateforme web multi-utilisateurs de segmentation video par IA locale. L'application tourne entierement dans Docker (3 conteneurs) et ne depend d'aucun service cloud — tout reste sur votre serveur.

Installation Docker

Prerequis

  • Docker >= 20.10 et Docker Compose >= 2.0
  • GPU recommande pour Ollama et Whisper (NVIDIA CUDA ou AMD ROCm) — fonctionne aussi en CPU, juste plus lent
  • ~20 GB d'espace (image Docker + modeles IA)

Lancement rapide

git clone https://github.com/King4Kats/Clipr.git
cd Clipr
docker compose up -d --build

L'app est accessible sur http://localhost:3000. Le premier utilisateur inscrit devient admin.

GPU NVIDIA

Decommentez la section deploy.resources dans docker-compose.yml pour le service ollama :

ollama:
  deploy:
    resources:
      reservations:
        devices:
          - capabilities: [gpu]
AMD (ROCm) : Ollama fonctionne mais Whisper tourne en CPU (int8). Pas de section deploy necessaire.

L'image est construite en 3 etapes :

  1. deps — Installe les node_modules (production only)
  2. build — Compile le client (Vite) et le serveur (TypeScript)

Pipeline IA

Endpoint : POST /api/project/:id/analyze

Le serveur execute le pipeline complet en arriere-plan :

1. Verifier le verrou IA (acquireLock) 2. Extraire l'audio de chaque video (FFmpeg → WAV 16kHz mono) 3. Transcrire chaque audio (faster-whisper + initial_prompt) 4. Analyser le transcript (Ollama: resume global → decoupe par chunks) 5. Nettoyer les segments (deduplication, sequentialisation, fusion) 6. Sauvegarder en DB + liberer le verrou 7. Broadcaster analysis:complete via WebSocket

Auto-save intermediaire apres chaque etape (audio, transcript, segments).

WebSocket

Connexion : ws://host/ws

Messages client → serveur

{"type": "auth", "token": "jwt-token"}       // Authentifier
{"type": "subscribe", "projectId": "uuid"}    // S'abonner a un projet
{"type": "unsubscribe"}                        // Se desabonner

{"type": "progress", "projectId": "...", "step": "transcribing", "progress": 45, "message": "..."}
{"type": "transcript:segment", "projectId": "...", "id": "0", "start": 1.2, "end": 3.5, "text": "..."}

Mise en production

Checklist avant deploiement

  1. JWT_SECRET — Definir une valeur aleatoire forte (32+ caracteres). Sans ca, les sessions sont perdues a chaque redemarrage du conteneur
  2. CORS_ORIGINS — Restreindre au domaine de production (ex: https://clipr.votre-domaine.fr)
  3. HTTPS — Configurer le Caddyfile avec votre domaine. Caddy gere les certificats automatiquement
  4. Sauvegardes — Planifier des backups reguliers du volume clipr-data (contient la DB SQLite et les fichiers)
  5. GPU — Activer si NVIDIA disponible pour accelerer Whisper et Ollama

Exemple docker-compose.yml production

services:
  clipr:
    build: .
    container_name: clipr
    ports:
      - "3000:3000"
    volumes:
      - clipr-data:/data
    environment:
      - JWT_SECRET=un-vrai-secret-aleatoire-de-32-caracteres
      - CORS_ORIGINS=https://clipr.votre-domaine.fr
      - OLLAMA_HOST=ollama
    depends_on:
      ollama:
        condition: service_healthy
    restart: always

Backup de la base

# Copier la DB SQLite depuis le conteneur
docker cp clipr:/data/clipr.db ./backup_clipr.db

# Ou monter un volume et sauvegarder depuis l'hote
docker compose exec clipr cp /data/clipr.db /data/backup_$(date +%Y%m%d).db

Pipeline — Transcription + Diarisation

Vue d'ensemble

Le pipeline de transcription audio/video execute les etapes suivantes en sequentiel (partage VRAM GPU) :

Audio/Video │ ▼ 1. FFmpeg ── Extraction audio (WAV 16kHz mono) │ Si video MTS → remux automatique en MP4 ▼ 2. Whisper ── Transcription (faster-whisper, GPU) │ Modele: large-v3 | Langue: fr │ → segments [{start, end, text}] │ → FREE GPU ▼ 3. Diarisation ── SpeechBrain ECAPA-TDNN (GPU) │ Sliding window 1.5s → embeddings → clustering │ → speaker labels par segment │ → FREE GPU ▼ 4. Ollama ── Detection des noms (analyse texte) │ Fallback: Intervenant / Intervieweur ▼ Save DB → transcription:complete (WebSocket) │ ▼ 5. Update DB ── Segments avec speakers → diarization-complete

Scripts Python

ScriptRoleGPU
scripts/transcribe.pyTranscription Whisper (single file)Oui
scripts/transcribe-batch.pyTranscription Whisper batch (N clips, modele charge 1 fois)Oui
scripts/diarize.pyDiarisation SpeechBrain (sliding window + clustering)Oui
scripts/phonetize.pyTranscription IPA via AllosaurusOui
scripts/whisperx-diarize.pyWhisperX tout-en-un (Whisper + pyannote + alignement)Oui

Tous les scripts suivent le meme protocole stderr :

PROGRESS: 42          // Pourcentage (0-100)
STATUS: Transcribing  // Message de statut
SEGMENT: {"id":0,...} // Segment en temps reel
ERROR: message        // Erreur

Gestion VRAM

Chaque script Python libere explicitement la VRAM GPU avant de quitter :

del model
torch.cuda.empty_cache()
gc.collect()

Le service Node attend 2 secondes apres chaque script (sleep 2) pour laisser le temps au driver CUDA de liberer la memoire.

Ne jamais lancer deux scripts GPU en parallele. Le pipeline est strictement sequentiel via la task queue (1 tache a la fois).

Pipeline — Transcription linguistique

Objectif

Documenter les langues vernaculaires. Un meneur dit une phrase en francais, puis 1 a 9 personnes la repetent dans leur dialecte.

Pipeline WhisperX

Audio │ ▼ 1. WhisperX ── Transcription + Diarisation + Alignement mot par mot │ Whisper (transcription) → FREE GPU │ Alignement (timestamps mots) → FREE GPU │ pyannote (diarisation) → FREE GPU │ → segments [{start, end, text, speaker, words}] ▼ 2. Identification meneur ── Premier speaker du fichier = meneur FR ▼ 3. Segmentation ── Tours meneur = sequences FR │ Tours autres = variantes vernaculaires │ Filtrage: sequences avec < 2 variantes supprimees ▼ 4. Allosaurus IPA ── Transcription phonetique des variantes (GPU) │ → FREE GPU ▼ 5. FFmpeg ── Decoupe clips audio par variante ▼ Save DB → linguistic:complete (WebSocket)

Upload chunked

Probleme

Le tunnel Cloudflare limite les requetes a ~100 Mo. Les fichiers video depassent souvent cette limite.

Solution

Cote client, les fichiers de plus de 90 Mo sont automatiquement decoupes en chunks :

Fichier > 90 Mo │ ▼ Client ── File.slice() → chunks de 90 Mo │ uploadId = timestamp + random │ ▼ (pour chaque chunk) POST /api/upload/chunk │ body: FormData {chunk, uploadId, chunkIndex, totalChunks, fileName} │ → stocke dans data/chunks/{uploadId}/chunk_00000 │ ▼ (apres le dernier chunk) POST /api/upload/chunk/complete │ body: {uploadId, fileName, totalChunks} │ → assemble les chunks en streaming │ → supprime le dossier chunks │ → ffprobe duration │ → retourne {id, path, name, duration, size} ▼ Fichier assemble dans data/uploads/

Cleanup automatique : les chunks abandonnes (> 1h) sont supprimes toutes les 30 minutes.

L'upload chunked est transparent pour l'utilisateur. Une barre de progression s'affiche avec le pourcentage.

Bienvenue sur Clipr

Plateforme de segmentation video par IA locale

Clipr segmente automatiquement des videos longues en extraits thematiques. Toutes les donnees restent sur votre serveur.

  • Multi-projets : jusqu'a 6 projets simultanes
  • Analyse en arriere-plan : lancez l'IA et continuez a travailler
  • Multi-utilisateurs : comptes avec authentification JWT
  • Collaboration : partagez vos projets (lecteur/editeur)
  • Administration : dashboard complet pour superviser l'instance

Installation

Prerequis

  • Docker >= 20.10 et Docker Compose >= 2.0
  • GPU recommande (NVIDIA CUDA ou AMD ROCm)
  • ~20 GB d'espace disque

Lancement

git clone https://github.com/King4Kats/Clipr.git
cd Clipr
docker compose up -d

Accessible sur http://localhost:3000

Creer un compte

  1. Ouvrir l'application
  2. Cliquer S'inscrire
  3. Remplir : nom d'utilisateur, email, mot de passe
Le premier utilisateur inscrit devient automatiquement administrateur.

Traitement en arriere-plan

Cliquer Lancer l'analyse. Le traitement s'execute cote serveur. Vous pouvez naviguer librement ou fermer le navigateur.

1. Extraction audio (FFmpeg) → WAV 16kHz mono 2. Transcription (Whisper) → Segments horodates 3. Analyse semantique (Ollama) → Segments thematiques 4. Sauvegarde en base → Statut "Termine"

Projet manuel

Ajoutez des segments manuellement sans IA : importez des videos, utilisez le bouton + dans la timeline, ajustez les timecodes et exportez.

Travaillez sur un projet manuel pendant qu'un projet IA s'analyse en arriere-plan.

Editeur

L'editeur NLE comprend : lecteur video, timeline, liste des segments (renommer, redimensionner, reordonner, supprimer) et transcription horodatee.

Pipeline IA

Endpoint : POST /api/project/:id/analyze

Le serveur execute le pipeline complet en arriere-plan :

1. Verifier le verrou IA (acquireLock) 2. Extraire l'audio de chaque video (FFmpeg → WAV 16kHz mono) 3. Transcrire chaque audio (faster-whisper + initial_prompt) 4. Analyser le transcript (Ollama: resume global → decoupe par chunks) 5. Nettoyer les segments (deduplication, sequentialisation, fusion) 6. Sauvegarder en DB + liberer le verrou 7. Broadcaster analysis:complete via WebSocket

Auto-save intermediaire apres chaque etape (audio, transcript, segments).

WebSocket

Connexion : ws://host/ws

Messages client → serveur

{"type": "auth", "token": "jwt-token"}       // Authentifier
{"type": "subscribe", "projectId": "uuid"}    // S'abonner a un projet
{"type": "unsubscribe"}                        // Se desabonner

Messages serveur → client

{"type": "progress", "projectId": "...", "step": "transcribing", "progress": 45, "message": "..."}
{"type": "transcript:segment", "projectId": "...", "id": "0", "start": 1.2, "end": 3.5, "text": "..."}
{"type": "analysis:complete", "projectId": "...", "segments": [...], "transcript": [...], "audioPaths": [...]}
{"type": "analysis:error", "projectId": "...", "message": "..."}
Le client doit envoyer auth avant subscribe. Les messages ne sont envoyes qu'aux clients abonnes au projet concerne.

Deploiement production

Checklist

  1. Definir JWT_SECRET avec une valeur aleatoire (32+ caracteres)
  2. Configurer CORS_ORIGINS avec votre domaine
  3. HTTPS via Caddy (automatique avec nom de domaine)
  4. Sauvegardes regulieres du volume clipr-data
services:
  clipr:
    environment:
      - JWT_SECRET=votre-secret-aleatoire-tres-long
      - CORS_ORIGINS=https://clipr.votre-domaine.fr

Developpement local

npm install          # Installer les dependances
npm run dev          # Client (Vite HMR) + Serveur (tsx watch)
npm run build        # Build production
npm start            # Lancer le serveur

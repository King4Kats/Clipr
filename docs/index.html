🎬 Bienvenue sur Clipr

Clipr est un logiciel de bureau qui analyse automatiquement vos videos grace a l'intelligence artificielle. Il ecoute le contenu, comprend les sujets abordes, et vous propose un decoupage en segments thematiques.

Concretement, vous importez une video (un cours, une conference, un podcast video...) et Clipr :

  1. Extrait l'audio de la video
  2. Transcrit les paroles en texte (grace a Whisper)
  3. Analyse le texte pour identifier les themes (grace a un LLM local)
  4. Vous propose un editeur pour ajuster les decoupages et exporter

Tout se passe en local sur votre ordinateur. Aucune donnee n'est envoyee sur Internet.

📷Capture d'ecran : ecran d'accueil de Clipr

💻 Installation

1 Telecharger l'installeur

Rendez-vous sur la page de telechargement et recuperez le fichier Clipr-Setup-X.Y.Z.exe.

2 Lancer l'installation

Double-cliquez sur le fichier telecharge. L'installation est automatique et ne necessite aucune configuration particuliere. Clipr s'installe dans votre profil utilisateur.

L'installeur ne necessite pas de droits administrateur.

3 Premier demarrage

Une fois installe, Clipr se lance automatiquement. Un assistant de configuration apparait pour telecharger les modeles d'IA necessaires.

📷Capture d'ecran : installeur Windows

⚙️ Premier lancement

Au premier demarrage, l'assistant de configuration (SetupWizard) vous guide pour preparer l'application :

Modeles IA a telecharger

  • Modele Whisper (~142 Mo) : le moteur de reconnaissance vocale qui transforme l'audio en texte
  • Modele LLM (~2.1 Go) : le cerveau qui analyse le texte et identifie les themes

Cliquez sur les boutons de telechargement et attendez que les barres de progression se remplissent. Une connexion Internet est necessaire pour cette etape uniquement.

Le telechargement du modele LLM peut prendre plusieurs minutes selon votre connexion.
📷Capture d'ecran : assistant de configuration

📁 Importer une video

Deux methodes pour importer :

  • Glisser-deposer : faites glisser votre fichier video directement dans la zone d'import
  • Clic : cliquez sur la zone d'import pour ouvrir l'explorateur de fichiers

Formats supportes

MP4 MOV AVI MKV MTS WebM

Vous pouvez importer plusieurs videos. Elles seront concatenees dans une timeline globale pour l'analyse.
📷Capture d'ecran : zone d'import

🤖 Lancer l'analyse IA

Apres l'import, un panneau de configuration apparait a droite de la video :

  • Modele Whisper : choisissez la precision (Base = rapide, Medium = meilleure qualite)
  • Langue : selectionnez la langue parlee dans la video
  • Modele LLM : le modele d'analyse (Qwen 3B par defaut)
  • Consignes : ajoutez des instructions pour guider l'IA (ex: "Cette video parle de cuisine italienne")

Cliquez sur "Lancer l'analyse". L'operation se deroule en 3 etapes automatiques :

🎵 Extraction audio → 🗣️ Transcription → 🧠 Analyse thematique
L'analyse peut durer de 2 a 30 minutes selon la longueur de la video et la puissance de votre machine.
📷Capture d'ecran : panneau d'analyse IA

✏️ Comprendre l'editeur

Une fois l'analyse terminee, vous accedez a l'editeur compose de 3 panneaux redimensionnables :

1. Lecteur video (en haut a gauche)

Previsualisez votre video. Quand un segment est selectionne, la lecture se limite a ce segment.

2. Liste des segments (en haut a droite)

Tous les segments identifies par l'IA avec leur titre, duree et couleur. Cliquez pour selectionner.

3. Timeline (en bas)

Vue chronologique avec la forme d'onde audio et les segments colores superposes. Zoomez avec Ctrl+Molette.

Les panneaux sont redimensionnables et deplacables. Utilisez le bouton de reinitialisation pour revenir a la disposition par defaut.
📷Capture d'ecran : editeur avec les 3 panneaux

✂️ Ajuster les segments

  • Renommer : cliquez sur le titre d'un segment pour le modifier
  • Deplacer les bornes : dans la timeline, glissez les bords d'un segment pour ajuster debut/fin
  • Ajouter : utilisez le bouton "+" pour creer un nouveau segment
  • Supprimer : cliquez sur l'icone poubelle d'un segment
  • Reordonner : glissez-deposez les segments dans la liste pour changer l'ordre

📥 Exporter

Le bouton d'export dans la liste des segments propose 3 options :

1. Videos decoupees (MP4)

Chaque segment est exporte comme un fichier video independant. Choisissez un dossier de destination.

2. Texte avec timecodes (TXT)

Un fichier texte contenant la transcription organisee par segment, avec les horaires de debut et fin.

3. Transcription propre (TXT)

Le texte brut de la transcription, sans timecodes, pret a etre copie ou reutilise.

🔄 Mise a jour

Clipr verifie automatiquement les mises a jour a chaque demarrage. Si une nouvelle version est disponible :

  1. Le telechargement se lance automatiquement en arriere-plan
  2. Un bouton "Redemarrer" apparait dans les parametres
  3. Cliquez dessus pour installer et relancer l'application
Si vous fermez l'application alors qu'une mise a jour est telechargee, elle s'installera automatiquement au prochain lancement.

🔎 Diagnostiquer un probleme

Si vous rencontrez un probleme, la section Diagnostic dans les parametres vous permet de :

Exporter les logs

Cliquez sur "Exporter les logs" pour generer un fichier ZIP contenant l'historique des actions de l'application. Transmettez ce fichier au support technique.

ID d'installation

Un identifiant unique (ex: a1b2c3d4) est affiche. Il permet de distinguer votre installation en cas de support multiple.

🏗️ Architecture Electron

Clipr est construit sur Electron, qui combine un processus Node.js (le Main Process) et un navigateur Chromium (le Renderer Process). La communication entre les deux passe par un pont securise : le Preload Script.

┌─────────────────────────────────────────────────────────────────┐ │ APPLICATION CLIPR │ │ │ │ ┌───────────────────────┐ IPC Bridge ┌──────────────────┐│ │ │ MAIN PROCESS │◄──────────────►│ RENDERER PROCESS ││ │ │ (Node.js) │ │ (Chromium/React) ││ │ │ │ │ ││ │ │ electron/main.ts │ preload.ts │ src/App.tsx ││ │ │ electron/services/* │◄──────────────►│ src/store/* ││ │ │ │ (pont IPC) │ src/components/*││ │ └───────────────────────┘ └──────────────────┘│ │ │ │ │ │ ▼ ▼ │ │ Acces systeme : Interface graphique : │ │ - Fichiers (fs) - React 18 + Tailwind │ │ - Processus (spawn) - Zustand (state) │ │ - FFmpeg, Python, Ollama - Framer Motion (anim) │ │ - electron-log, updater - Radix UI (composants) │ └─────────────────────────────────────────────────────────────────┘

Preloadelectron/preload.tsPont securise. Expose une API limitee (window.electron) au renderer via contextBridge. Aucun acces direct a Node.js depuis le renderer.

⚙️ Pipeline de traitement

Le pipeline est orchestre par la methode triggerAnalysis() du store Zustand. C'est une sequence de 3 etapes asynchrones avec gestion d'erreur globale.

triggerAnalysis() dans useStore.ts │ ├─ ETAPE 1 : Extraction Audio │ │ processingStep = 'extracting-audio' │ │ │ │ Pour chaque videoFile : │ │ └─ IPC: ffmpeg:extractAudio(videoPath) │ │ └─ FFmpeg convertit en WAV 16kHz mono │ │ └─ Progress via IPC: processing:progress │ │ │ └─ Resultat : audioPaths[] (fichiers temporaires) │ ├─ ETAPE 2 : Transcription │ │ processingStep = 'transcribing' │ │ │ │ Pour chaque audioPath (sequentiel) : │ │ └─ IPC: whisper:transcribe(audioPath, language) │ │ └─ Python faster-whisper process │ │ └─ Segments envoyes en temps reel via whisper:segment │ │ └─ Timestamps ajustes avec l'offset video │ │ │ └─ Resultat : transcript[] (TranscriptSegment[]) │ ├─ ETAPE 3 : Analyse Thematique │ │ processingStep = 'analyzing' │ │ │ │ 1. ensureOllamaRunning() - Demarre le serveur si besoin │ │ 2. generateGlobalSummary() - Resume general du contenu │ │ 3. splitTranscriptIntoChunks() - Decoupe en morceaux de ~30K chars │ │ 4. Pour chaque chunk : │ │ └─ analyzeChunk() - Ollama identifie les segments │ │ └─ Prompt systeme strict (JSON, durees, titres) │ │ └─ Retry loop (3 tentatives, 2s delai) │ │ 5. cleanAndForceSequential() - Post-traitement : │ │ - Deduplication (merge si >70% overlap) │ │ - Couverture totale (debut=0, fin=duree) │ │ - Segments courts absorbes (<30s) │ │ │ └─ Resultat : segments[] (VideoSegment[]) │ └─ FINALISATION │ processingStep = 'ready' │ setSegments(segments) - Couleurs auto-assignees │ autoSave() - Sauvegarde du projet │ └─ L'UI bascule vers l'EditorLayout

Multi-video : Timeline unifiee

Quand plusieurs videos sont importees, elles sont concatenees dans une timeline globale grace a un systeme d'offsets :

Video 1 : offset=0,    duree=120s  → [0s   - 120s]
Video 2 : offset=120,  duree=90s   → [120s - 210s]
Video 3 : offset=210,  duree=150s  → [210s - 360s]
                                      ↑ Timeline globale

Les methodes utilitaires du store :

  • getVideoAtTime(globalTime) → retourne {video, localTime, index}
  • getClipsForSegment(start, end) → retourne les VideoClip[] pour l'export (un segment peut chevaucher plusieurs videos)
  • getTotalDuration() → somme des durees de toutes les videos

📊 State Management (Zustand)

🔧 Services Backend

Les services sont des modules Node.js isoles dans electron/services/. Chacun a une responsabilite unique et communique avec le renderer via les handlers IPC de main.ts.

ffmpeg.ts Traitement video

Wrapper autour de FFmpeg/FFprobe. Gere l'initialisation des binaires (packages, npm, systeme), l'extraction audio, le decoupage et la concatenation.

FonctionDescription
initFFmpeg()Localise les binaires : packages (production) → @ffmpeg-installer (dev) → PATH systeme
checkFFmpeg()Verifie la disponibilite de FFmpeg
getVideoDuration(path)Recupere la duree via ffprobe -print_format json (spawn manuel avec windowsHide: true)
extractAudio(path, onProgress)Convertit en WAV PCM 16kHz mono (format Whisper)
cutVideo(input, start, end, output, quality)Decoupe avec libx264 + AAC, CRF ajustable
concatenateVideos(paths, output)Fusionne plusieurs videos via mergeToFile()
windowsHide : toutes les commandes utilisent windowsHide: true pour empecher le flash CMD sur Windows. Un helper hideWindow() applique cette option sur les commandes fluent-ffmpeg.

whisper.ts Transcription vocale

Interface avec le moteur faster-whisper via un subprocess Python. Le script transcribe.py est fourni avec l'application.

FonctionDescription
loadWhisperModel(model)Selectionne le modele (tiny/base/small/medium)
transcribe(audioPath, lang, onSegment, onProgress)Lance la transcription. Les segments arrivent en temps reel via stderr
cancelTranscription()Tue le processus Python en cours

Protocole stderr : le script Python communique via des prefixes sur stderr :

PROGRESS:45          → Pourcentage de progression
STATUS:Loading model  → Message de statut
SEGMENT:{"id":...}   → Segment transcrit (JSON)
WHISPER_ENGINE:info   → Info moteur
ERROR:message         → Erreur

ollama.ts Analyse LLM

Communique avec le serveur Ollama local (port 11434) via HTTP. Gere le demarrage automatique, l'analyse par chunks et le post-traitement des segments.

FonctionDescription
checkOllama()Teste si le serveur repond (GET /api/tags)
ensureOllamaRunning()Demarre ollama serve si necessaire, poll 20x500ms
downloadAndInstallOllama(onProgress)Telecharge OllamaSetup.exe depuis ollama.com et l'execute
listOllamaModels()Liste les modeles installes
pullOllamaModel(modelName)Telecharge un modele via POST /api/pull
analyzeTranscript(transcript, context, model, onProgress)Pipeline complet : resume global → chunks → analyse → nettoyage
generateGlobalSummary(transcript, model)Cree un resume narratif pour contextualiser l'analyse
splitTranscriptIntoChunks(transcript)Decoupe en morceaux de ~30K chars avec 800 chars d'overlap
analyzeChunk(chunk, context, summary, model, ...)Analyse un chunk avec prompt strict (JSON, durees, titres). Retry x3, delai 2s.
cleanAndForceSequential(segments, duration)Post-traitement : deduplication (>70% overlap), couverture totale, merge courts (<30s)
mergeAndDeduplicateSegments(allSegments)Fusion des segments provenant de plusieurs chunks (multi-chunk)
generateSegmentTitle(transcript, model)Genere un titre pour un segment via le LLM

Chunking : les transcriptions longues sont decoupees en morceaux de ~30 000 caracteres avec un chevauchement de 800 caracteres pour la continuite. Chaque chunk recoit le dernier segment du chunk precedent pour assurer la sequentialite.

Toutes les interfaces sont definies dans src/types/index.ts.

interface VideoFile {
  path: string       // Chemin complet du fichier
  name: string       // Nom du fichier
  duration: number   // Duree en secondes
  size: number       // Taille en octets
  offset: number     // Position dans la timeline globale (auto-calcule)
}

interface TranscriptSegment {
  id: string
  start: number      // Debut en secondes
  end: number        // Fin en secondes
  text: string       // Texte transcrit
}

interface VideoSegment {
  id: string
  title: string
  start: number      // Debut dans la timeline globale
  end: number        // Fin dans la timeline globale
  transcriptSegments: TranscriptSegment[]
  color: string      // Couleur hex (auto-assignee)
}

interface VideoClip {
  videoIndex: number
  videoPath: string
  start: number      // Temps local dans cette video
  end: number        // Temps local dans cette video
}

interface AppConfig {
  whisperModel: 'tiny' | 'base' | 'small' | 'medium' | 'large'
  ollamaModel: string
  language: string
  outputQuality: number   // 0-51, plus bas = meilleure qualite
  outputFolder: string | null
  context: string         // Consignes utilisateur pour l'IA
}

type ProcessingStep =
  | 'idle' | 'extracting-audio' | 'transcribing'
  | 'analyzing' | 'ready' | 'exporting' | 'done' | 'error'

type UpdateStatus =
  | { status: 'checking' }
  | { status: 'available'; version: string; releaseNotes?: string }
  | { status: 'not-available'; version: string }
  | { status: 'downloading'; percent: number; bytesPerSecond: number;
      transferred: number; total: number }
  | { status: 'downloaded'; version: string }
  | { status: 'error'; message: string }

interface ModelStatus {
  whisper: { model: string; downloaded: boolean }

interface DependencyStatus {
  name: string

Export des logs

Le service log-sender.ts gere l'export :

  1. Au premier lancement, un UUID v4 est genere et stocke dans installation-id.json
  2. L'export cree un ZIP (compression niveau 9) contenant les logs + l'UUID
  3. L'utilisateur choisit l'emplacement via un dialogue "Enregistrer sous"
clipr-logs-a1b2c3d4.zip
├── logs/
│   ├── main.log           # Log actuel
│   └── main.old.log       # Log precedent (rotation)
└── installation-id.json   # UUID unique de l'installation

🔄 Mise a jour automatique

Le systeme utilise electron-updater avec GitHub Releases comme source.

App demarre │ ▼ Attente 5 sec (evite de ralentir le demarrage) │ ▼ Verification GitHub GET .../releases/latest → latest.yml │ ┌─┴─────────────┐ ▼ ▼ A jour MAJ disponible │ ▼ Telechargement (arriere-plan, barre de progression) │ ▼ Pret a installer → Bouton "Redemarrer" │ (clic utilisateur) │ ▼ quitAndInstall() → Ferme, installe, relance

Configuration

autoUpdater.autoDownload = false       // Controle manuel du telechargement
autoUpdater.autoInstallOnAppQuit = true // Installe a la fermeture si MAJ prete

// Source (package.json → build.publish)
{
  "provider": "github",
  "owner": "King4Kats",
  "repo": "Clipr"
}
Important : Le fichier latest.yml genere par electron-builder est indispensable. Sans lui, electron-updater ne peut pas detecter les mises a jour. Toujours le publier avec la release.
